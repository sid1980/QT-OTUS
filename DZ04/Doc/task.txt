Работа с базой данных
Цель:

    Потренироваться в работе с базой данных
    Потренироваться в работе с моделями

Описание/Пошаговая инструкция выполнения домашнего задания:

Как и предидущее ДЗ, это позволяет поставить собственную задачу и ее выполнить или выбрать из существующих вариантов.
Требуется написать приложение которое будет работать с какими либо упорядоченными данными (таблица/дерево).
Идеально если задача подразумевает использование кастомных делегатов и работы с разнородными данными (которые можно отобразить в таблице и те которые туда не помещаются.
Стандартная постановка
Нужно:

    Создать приложение Qt Widgets с главным окном (наследником QMainWindow)
    Создать в menubar главного окна меню File или Db с пунктом connect. По нажатию на пункту connect должно будет происходить подключение к источнику данных
    Поместить на главном окне нужный вам View и другие нужные вам элементы управления/отображения
    Установить нужные вам взаимосвязи View-Model-Delegate, а также если требуется их взаимосвязь с другими элементами управления
    Добиться требуемой визуализации
    Предопределенные кейсы:
    https://cloud.mail.ru/public/ZANg/UKCZvcVhj
    gold.csv - данные по курсу золота с 2009 по 2020 год. Написать приложение которое визуализирует данные в виде таблицы которое в ячейках цены показывает кроме самого значения показывает движение цены (вверх/вниз/без изменений, визуализировать можно стрелочками цветом и любым другим способом который посчитаете нужным)
    INTFETAL.sqlite - медицинские данные пациентов. Отображать данные по пациентам в виде таблицы и карточки. В базе нет данных по ЕКГ, но в качестве задания со * можно нагенерировать данные и также обобразить их в интерфейсе
    cppref_db.sqlite - документация по с++. Отображать данные по алгоритмам С++. Сводную информацию отображать в виде таблицы. Информацию семантике методов в отдельном виджете (TextBrowser например). Стандарты в таблице отображать в виде иконок. Информации по семантике методов в базе нет, но можно скачать несколько html с cppreference и использовать их или завести отдельную таблицу и добавить туда пример описания.


Критерии оценки:

    Приложение собирается
    Приложение функционирует
    Приложение соответсвует первоначальной задумке
